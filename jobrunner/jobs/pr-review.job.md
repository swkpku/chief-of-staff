# PR Review

## Schedule
0 9 * * 1-5

## Goal
Review open pull requests on our main repositories.
Check for code quality issues, potential bugs, and style consistency.
Leave comments on PRs that need attention and approve clean ones.

## Policy
- Approve PRs that are documentation-only or minor formatting fixes
- Comment on PRs with potential bugs or security issues
- Flag PRs that change more than 500 lines for manual review
- Summarize each PR review with key findings

## Boundaries
- Never merge a PR without approval
- Never close a PR
- Never force-push or modify PR branches
